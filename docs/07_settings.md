# 07 Settings Module

The Settings layer determines how documents are named, sized, and compressed before export.

## Smart Naming Rules
Names are automatically generated by the backend API (`/api/process-image`). The frontend only collects the parameters.
**User input "Name" has been removed.**

Format: `<relationship>_<document_type>[_<custom_suffix>].<extension>`

- **Relationship**: Self, Mother, Father, Brother, Sister, Spouse, Child.
- **Document Type**: Inherited from the initial workflow selection (e.g., Aadhaar, PAN).
- **Custom Suffix**: Optional (e.g., "front", "back").

**Examples:**
- `self_aadhaar.jpg`
- `mother_pan.webp`
- `brother_passport_front.png`

*All spaces are replaced with underscores, and the entire filename is converted to lowercase automatically.*

## Output Settings
These settings are collected by the frontend and executed by the Python backend.

### Resolution (Resize)
1. Original Size
2. Preset: 160 × 200 (Passport)
3. Preset: 300 × 300
4. Preset: 600 × 600
5. Custom Width × Height

### Target File Size (Compression)
1. No Compression Limit
2. Preset: Under 19 KB
3. Preset: Under 50 KB
4. Preset: Under 100 KB
5. Custom Target KB

The backend uses a binary-search algorithm adjusting JPEG/WEBP quality iteratively (up to 7 passes) to accurately hit the target file size.

### Format
1. JPG (Best for compression)
2. PNG (Lossless, larger file)
3. WEBP (Modern, efficient)
